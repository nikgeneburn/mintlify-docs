Every drawable in prim32 — rect, rounded rect, circle, ring, line, glyph, image, drop shadow —
is **one 32-byte GPU primitive**. Widgets write those primitives straight into
persistently-mapped GPU memory, the vertex shader expands them into quads, and the whole
UI renders in about one draw call per layer.

```cpp
prim32::FontHandle mainFont = prim32::LoadFontFromFile("assets/Inter-Regular.ttf", 16.0f);
prim32::SetDefaultFont(mainFont);
prim32::ImageHandle logo = prim32::LoadImageFromFile("assets/logo.png");

prim32::DrawList* draw = prim32::GetDrawList();

draw->Text("Hello from the default font", {20, 20});
draw->Text(mainFont, "Or an explicit font", {20, 50});
draw->Image(logo, prim32::Rect{20, 90, 128, 128});
```

## Measured, not promised

Numbers from the included benchmark (RTX 3060, 1680×945, i7 16-core):

| What | Result |
|---|---|
| CPU emit, scalar | ~275 M prims/s per core (3.6 ns/prim) |
| CPU emit, SIMD bulk path | ~850 M prims/s (1.2 ns/prim, SSE2 non-temporal streams) |
| 2,000,000 animated rounded rects | 234 fps — GPU raster is the wall, CPU at 0.12 ms |
| 95,000 rounded rects | 0.55 ms GPU / frame |
| Cached layer redraw | zero CPU emit, zero PCIe traffic, movement via one root constant |
| Profiler scope overhead | ~30 ns |
